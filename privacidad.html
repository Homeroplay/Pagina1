03. Alojamiento y Publicación: Poner tu Sitio en Línea

El alojamiento web es el servicio que guarda tus archivos en un servidor para que estén disponibles 24/7 en internet. La publicación es el proceso de subir tu contenido y hacerlo accesible al público.

¿Qué es el Alojamiento Web?

El alojamiento web (web hosting) es un servicio donde alquilas espacio en un servidor para almacenar los archivos de tu sitio web. Estos servidores están conectados permanentemente a internet y optimizados para servir páginas web rápidamente.

Diferencia entre Dominio y Alojamiento

Dominio: Es la dirección web (ejemplo.com) — lo que escribes en el navegador

Alojamiento: Es donde viven tus archivos — el espacio en el servidor

Analogía: El dominio es la dirección de tu casa, el alojamiento es la casa misma

Pasos para Publicar tu Sitio Web

Paso 1: Elegir un Proveedor de Alojamiento

Busca un host confiable que ofrezca:

  • Espacio disco suficiente para tu sitio
  • Ancho de banda ilimitado o generoso
  • Uptime garantizado del 99.9%
  • Soporte técnico 24/7
  • Certificado SSL gratuito (HTTPS)
  • Panel de control intuitivo (cPanel, Plesk)

Paso 2: Contratar el Plan de Hosting

Los planes típicos son:

  • Shared Hosting: $5-15/mes, ideal para principiantes
  • VPS: $15-50/mes, mejor para sitios en crecimiento
  • Dedicated: $50-200+/mes, para sitios muy grandes
  • Cloud: Variable, escalable según uso

Paso 3: Conectar tu Dominio

Hay dos métodos:

  • Cambiar los Nameservers: Apunta tu dominio a los servidores de nombres del hosting
  • Configurar Registros A/CNAME: Vincula directamente la IP del servidor a tu dominio

Nota: La propagación de DNS puede tardar 24-48 horas

Paso 4: Acceder al Panel de Control

Usa tus credenciales para entrar al cPanel/Plesk donde puedes:

  • Crear cuentas de email
  • Gestionar archivos
  • Configurar bases de datos
  • Ver estadísticas de tráfico
  • Instalar aplicaciones (WordPress, Joomla, etc.)

Paso 5: Subir tus Archivos

Puedes subir archivos de varias formas:

  • FTP (File Transfer Protocol): Herramienta clásica, necesita cliente FTP
  • SFTP (FTP Seguro): Versión encriptada de FTP
  • File Manager Web: Interfaz gráfica en el cPanel
  • Git: Para desarrolladores, control de versiones

Paso 6: Configurar Base de Datos (si la necesitas)

Para aplicaciones dinámicas como WordPress:

  • Crea una base de datos MySQL o MariaDB
  • Crea un usuario con permisos
  • Anota las credenciales para la instalación

Paso 7: Instalar SSL/HTTPS

La mayoría de hosting ofrece Let's Encrypt gratuito:

  • Instala el certificado SSL desde el cPanel
  • Fuerza redirección HTTPS automática
  • Verifica que el candado aparece en el navegador

Paso 8: Optimizar y Probar

Antes de considerar finalizado:

  • Prueba en diferentes navegadores y dispositivos
  • Verifica velocidad de carga
  • Comprueba que todos los links funcionen
  • Configura copias de seguridad automáticas
  • Instala plugins de seguridad (si usas WordPress)

Herramientas para Subir Archivos (FTP)

Clientes FTP Recomendados:

  • FileZilla: Gratuito, multiplataforma, muy popular
  • WinSCP: Para Windows, interfaz intuitiva
  • Cyberduck: Para Mac, elegante y fácil
  • VS Code Remote: Para desarrolladores

Publicación con CMS (Content Management System)

Si no quieres tocar código, usa un CMS:

  • WordPress: El más popular, flexible, miles de plugins
  • Joomla: Potente, para sitios complejos
  • Drupal: Robusto, para aplicaciones grandes
  • Ghost: Minimalista, excelente para blogs
  • Wix / Squarespace: Constructores visuales sin código

Mantenimiento Post-Publicación

Tareas Regulares:

  • Actualizar plugins y temas mensualmente
  • Realizar copias de seguridad semanales
  • Monitorear rendimiento y analíticas
  • Revisar logs de error
  • Actualizar contenido regularmente
  • Verificar links rotos
  • Renovar certificados SSL anualmente

Errores Comunes en la Publicación

  • ❌ No hacer copias de seguridad antes de cambios importantes
  • ❌ Confundir los archivos raíz (public_html o www)
  • ❌ No usar HTTPS/SSL
  • ❌ Subir la carpeta incorrecta al servidor
  • ❌ Olvidar que archivos .htaccess pueden dañar el sitio
  • ❌ No actualizar plugins y temas regularmente
  • ❌ No verificar permisos de archivos (chmod)

Checklist Final de Publicación

  • ☑️ Dominio registrado y conectado
  • ☑️ Hosting contratado y activo
  • ☑️ Archivos subidos correctamente
  • ☑️ Base de datos configurada (si aplica)
  • ☑️ SSL/HTTPS instalado
  • ☑️ Sitio visible en navegador
  • ☑️ Enlaces funcionales verificados
  • ☑️ Velocidad de carga optimizada
  • ☑️ Copias de seguridad automáticas activas
  • ☑️ Email del dominio funcionando
